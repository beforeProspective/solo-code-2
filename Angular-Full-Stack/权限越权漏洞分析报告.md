# 权限越权漏洞分析报告

## 漏洞概述

本系统存在严重的**垂直权限越权漏洞**，允许普通用户在注册和更新个人信息时直接将自己的角色提升为管理员（admin），从而获取系统最高权限。

## 二、越权风险分析

### 2.1 漏洞类型
**垂直权限越权（Privilege Escalation）** - 低权限用户获取高权限角色

### 2.2 风险等级
**严重（Critical）**

### 2.3 攻击利用方式

#### 方式一：注册时直接选择 admin 角色
1. 攻击者访问注册页面 `/register`
2. 在表单的 Role 下拉框中直接选择 "Admin"
3. 填写其他必要信息后提交注册
4. 后端直接保存包含 `role: "admin"` 的用户数据
5. 攻击者使用新注册的账号登录，获得管理员权限

#### 方式二：普通用户登录后修改自身角色
1. 攻击者使用普通用户账号登录
2. 访问个人账户设置页面 `/account`
3. 在 Role 下拉框中将角色从 "User" 改为 "Admin"
4. 点击保存，前端发送包含 `role: "admin"` 的 PUT 请求
5. 后端直接更新用户角色为 admin
6. 系统返回新的 token，攻击者立即获得管理员权限

#### 方式三：不通过前端UI，直接调用API
即使前端移除了 role 选择框，攻击者仍可通过以下方式利用：

```bash
# 注册时直接提权
curl -X POST http://localhost:3000/api/user \
  -H "Content-Type: application/json" \
  -d '{"username":"hacker","email":"hacker@test.com","password":"123456","role":"admin"}'

# 登录后修改自身角色
curl -X PUT http://localhost:3000/api/user/[user_id] \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer [user_token]" \
  -d '{"role":"admin"}'
```

### 2.4 潜在危害

1. **数据泄露**：管理员可访问所有用户数据、系统配置等敏感信息
2. **数据篡改**：可任意修改或删除系统中的数据
3. **账户接管**：可创建、修改、删除其他用户账户
4. **系统破坏**：可执行破坏性操作，如删除整个数据库
5. **进一步渗透**：以管理员身份为跳板，进行更深层次的攻击

## 三、修复方案

### 3.1 后端修复（核心修复）

#### 方案概述
后端必须作为安全边界，对所有输入进行严格验证和过滤。

#### 修复步骤

**1. 用户注册时强制设置默认角色**

修改 [base.ts](file:///e:/solo-code-2/Angular-Full-Stack/server/controllers/base.ts) 的 `insert` 方法，或在 [user.ts](file:///e:/solo-code-2/Angular-Full-Stack/server/controllers/user.ts) 中覆写 `insert` 方法：

```typescript
// 在 user.ts 中添加 insert 方法覆写
insert = async (req: Request, res: Response) => {
  try {
    // 过滤请求体，只允许注册必要字段，强制 role 为 user
    const userData = {
      username: req.body.username,
      email: req.body.email,
      password: req.body.password,
      role: 'user' // 强制设置为普通用户，忽略前端传来的 role
    };
    
    const obj = await new this.model(userData).save();
    return res.status(201).json(obj);
  } catch (err) {
    return res.status(400).json({ error: (err as Error).message });
  }
};
```

**2. 用户更新时进行权限验证和字段过滤**

修改 [user.ts](file:///e:/solo-code-2/Angular-Full-Stack/server/controllers/user.ts) 的 `update` 方法：

```typescript
update = async (req: Request, res: Response) => {
  try {
    // 1. 从 token 中获取当前登录用户信息（需要先实现认证中间件）
    const currentUser = req.user; // 假设认证中间件已将用户信息挂载到 req.user
    
    // 2. 权限检查：只有 admin 才能修改 role 字段
    const updateData: any = {
      username: req.body.username,
      email: req.body.email,
    };
    
    // 3. 仅当当前用户是 admin 时，才允许修改 role
    if (currentUser && currentUser.role === 'admin') {
      updateData.role = req.body.role;
    }
    
    // 4. 如果用户修改自己的密码，需要额外验证（可选）
    if (req.body.password) {
      updateData.password = req.body.password;
    }
    
    const user = await this.model.findOneAndUpdate(
      { _id: req.params.id }, 
      updateData, 
      { new: true }
    );
    
    if (!user) {
      return res.sendStatus(404);
    }
    
    const token = sign({ user }, secret, { expiresIn: '24h' });
    return res.status(200).json({ token });
  } catch (err) {
    return res.status(400).json({ error: (err as Error).message });
  }
};
```

**3. 添加认证中间件**

创建认证中间件来验证 token 并挂载用户信息：

```typescript
// server/middleware/auth.ts
import { Request, Response, NextFunction } from 'express';
import { verify, Secret } from 'jsonwebtoken';

const secret: Secret = process.env.SECRET_TOKEN as string;

export const authMiddleware = (req: Request, res: Response, next: NextFunction) => {
  const token = req.headers.authorization?.split(' ')[1];
  
  if (!token) {
    return res.sendStatus(401);
  }
  
  try {
    const decoded = verify(token, secret) as any;
    req.user = decoded.user;
    next();
  } catch (err) {
    return res.sendStatus(401);
  }
};
```

**4. 在路由中应用认证中间件**

修改 [routes.ts](file:///e:/solo-code-2/Angular-Full-Stack/server/routes.ts)：

```typescript
import { authMiddleware } from './middleware/auth';

// 需要认证的路由
router.route('/user/:id').get(authMiddleware, userCtrl.get);
router.route('/user/:id').put(authMiddleware, userCtrl.update);
router.route('/user/:id').delete(authMiddleware, userCtrl.delete);
router.route('/users').get(authMiddleware, userCtrl.getAll);
router.route('/users/count').get(authMiddleware, userCtrl.count);
```

**5. 添加管理员权限中间件（可选）**

```typescript
export const adminMiddleware = (req: Request, res: Response, next: NextFunction) => {
  if (req.user && req.user.role === 'admin') {
    next();
  } else {
    return res.sendStatus(403);
  }
};
```

## 四、总结

### 根本原因
1. 前端直接暴露 `role` 字段给用户修改
2. 后端完全信任前端传入的数据，未进行任何权限验证和字段过滤
3. 缺少认证中间件，无法识别请求者的身份和权限

### 修复原则
1. **永不信任前端**：所有安全检查必须在后端进行
2. **最小权限原则**：默认授予最低权限，需要时才提升
3. **显式权限验证**：每个敏感操作都必须验证用户权限
4. **输入白名单**：只允许更新明确列出的字段，拒绝未授权的字段

### 建议
1. 立即实施上述后端修复方案
2. 添加单元测试验证权限控制逻辑
3. 定期进行安全代码审查
4. 考虑使用成熟的权限管理库（如 CASL、ngx-permissions 等）
